# 大学小组作业协作指南

## 小组作业常见问题

- 有人不干活（free rider）
- 分工不均
- 沟通不畅导致截止日期前才赶工
- 观点分歧无法统一

## 高效协作方法

### 第一步：明确分工（第一次小组会议）

| 角色 | 职责 |
|------|------|
| 组长 | 统筹进度、对外沟通（和老师确认要求） |
| 资料收集 | 查找文献、数据、案例 |
| 内容撰写 | 按章节分头写，最后由一人统稿 |
| PPT/展示 | 制作演示材料，准备演讲 |
| 质控 | 检查逻辑、格式、引用 |

全员都要参与，不要出现"只挂名不干活"的组员。

### 第二步：设定时间线

- 项目启动：收到任务后 24 小时内开第一次会
- 初稿：截止前 5-7 天完成各自部分
- 统稿：截止前 3-5 天
- 定稿：截止前 1-2 天

### 第三步：使用协作工具

- 共享文档：腾讯文档、石墨文档（实时编辑+评论）
- 文件共享：百度网盘、阿里云盘
- 即时沟通：微信群（建立专门的项目群）
- 任务管理：飞书文档的任务列表、Notion

### 应对不干活的组员

- 第一次私下提醒
- 第二次在组内公开说
- 第三次告知老师（大部分老师会酌情调整给分）
- 保留每次讨论和分工的聊天记录作为证据
